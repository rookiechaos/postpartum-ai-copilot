# 安全指南

## 概述

本文档描述了 Postpartum AI Copilot 的安全措施和最佳实践。

## 安全措施

### 1. 认证和授权

#### JWT Token 安全
- 使用 HS256 算法进行签名
- Token 包含过期时间
- 支持刷新 Token 机制
- Secret key 必须从环境变量读取，不能使用默认值

#### 密码安全
- 使用 pbkdf2_sha256 进行密码哈希
- 密码最小长度：8 字符
- 密码强度验证（包含大小写字母、数字）
- 密码验证错误不泄露具体信息

### 2. 输入验证

#### 数据验证
- 所有用户输入通过 Pydantic 模型验证
- 字符串长度限制
- 邮箱格式验证
- SQL 注入防护（通过 SQLAlchemy ORM）

#### 内容安全
- AI 建议经过多层验证
- 医疗安全规则检查
- 危险内容过滤

### 3. 数据保护

#### 敏感数据
- 密码：仅存储哈希值
- API 密钥：仅从环境变量读取
- 用户数据：访问控制确保用户只能访问自己的数据

#### 错误处理
- 生产环境不泄露堆栈跟踪
- 错误信息不包含敏感信息
- 统一错误处理中间件

### 4. API 安全

#### CORS 配置
- 限制允许的源
- 不允许通配符（生产环境）

#### 速率限制
- 使用 slowapi 实现速率限制
- 不同端点有不同的限制

#### 安全头
- 建议添加安全响应头（CSP, HSTS, X-Frame-Options等）

### 5. 配置安全

#### 环境变量
- 敏感配置从环境变量读取
- 使用 .env 文件（不提交到版本控制）
- 生产环境使用强密钥

#### 默认值
- 所有默认值仅用于开发环境
- 生产环境必须设置所有安全相关配置

## 安全检查清单

### 部署前检查

- [ ] JWT_SECRET_KEY 已设置为强随机值（至少32字符，不是默认值）
- [ ] 所有 API 密钥已设置
- [ ] CORS_ORIGINS 已正确配置（不包含通配符 `*`）
- [ ] DEBUG 模式已关闭
- [ ] 数据库连接使用安全凭据
- [ ] 所有默认密码已更改
- [ ] STRIPE_WEBHOOK_SECRET 已配置（如果使用Stripe）
- [ ] PAYPAL_WEBHOOK_ID 已配置（如果使用PayPal）
- [ ] MAX_UPLOAD_SIZE_MB 已根据需求配置（默认10MB）

### 运行时检查

- [ ] 速率限制正常工作
- [ ] 认证中间件正常工作
- [ ] 错误处理不泄露敏感信息
- [ ] 日志不包含敏感信息
- [ ] 用户数据访问控制正常

## 安全最佳实践

### 1. 密码管理
- 使用强密码策略
- 定期更新密码
- 不在代码中硬编码密码

### 2. Token 管理
- Token 过期时间合理（24小时）
- 刷新 Token 过期时间较长（30天）
- 支持 Token 撤销

### 3. 数据访问
- 始终验证用户身份
- 检查用户权限
- 使用参数化查询（SQLAlchemy 自动处理）

### 4. 日志记录
- 记录安全事件
- 不记录敏感信息
- 定期审查日志

### 5. 依赖管理
- 定期更新依赖
- 检查已知漏洞
- 使用可信的包源

## 安全响应

### 发现漏洞
1. 立即报告给安全团队
2. 评估影响范围
3. 制定修复计划
4. 实施修复
5. 验证修复
6. 通知受影响用户（如需要）

### 安全事件
1. 立即隔离受影响系统
2. 收集证据
3. 分析攻击向量
4. 修复漏洞
5. 恢复服务
6. 事后审查

## 合规性

### 数据保护
- 遵循 GDPR 原则（如适用）
- 用户数据最小化
- 数据加密传输
- 数据访问审计

### 医疗合规
- 本产品为陪伴与支持工具，非医疗产品、非医疗设备，不替代专业医疗或心理健康服务。
- 不提供医疗诊断
- 所有建议包含免责声明
- 紧急情况引导用户就医
- 记录所有交互（用于审计）
